Common workflow

Use files for longer text, keep the browser one flag away.

The main create and read flows support --body-file, --notes-file, and --web so you can move between terminal and browser without extra glue.

Example session
$ gc repo create demo-repo --public --clone --ssh
✓ Created https://gitcode.com/andy/demo-repo
✓ Cloned git@gitcode.com:andy/demo-repo.git

$ gc issue create andy/demo-repo --title "Crash on launch" --body-file ./issue.md
✓ Opened issue #12

$ gc pr create andy/demo-repo --title "Fix startup path" --body-file ./pr.md --base main --head fix/startup
✓ Created pull request !7

$ gc pr view 7 --web
✓ Opened https://gitcode.com/andy/demo-repo/pulls/7

$ gc release create andy/demo-repo --tag v1.2.3 --title "v1.2.3" --notes-file ./release-notes.md
✓ Published release v1.2.3
